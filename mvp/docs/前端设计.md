## 0. 技术栈与风格基调

| 项 | 选型 |
|---|---|
| 框架 | **React 18 + Vite** |
| 样式 | **TailwindCSS**(Utility-first,配合自定义 design tokens) |
| 字体 | **Inter**(正文/UI)+ **Source Serif 4**(标题/重点强调) |
| 视觉基调 | **浅色系**,大量留白,信息分层清晰,对标 **Claude.ai** |
| 主色 | 见 §0.1 设计令牌 |
| 路由 | React Router v6 |
| 状态管理 | Zustand(轻量,够用)或 React Context |
| HTTP/SSE | fetch + EventSource(SSE 原生 API) |
| 动画 | Framer Motion(用于打字机、彩蛋丝带、状态切换) |
| 图表 | Recharts(报告页雷达图) |

## 1. 整体页面结构

**多页 Web 应用**,4 个核心页面 + 1 个全局调试视图:

```
/                       首页(JD 投递 + 产品介绍 + Bento Grid)
/prepare                准备页(选人格 + 调滑块 + 简历占位)
/interview/:sessionId   面试房间(SSE 实时对话)
/report/:sessionId      面试报告(评级 + 雷达图 + 逐题回放)

[全局 Debug Drawer]     从右侧抽屉滑出,任意页面可激活(URL ?debug=1 或快捷键)
```

### 3.2 人格卡片清单

| ID | 卡片名 | 标签 | 默认滑块(温/深/节/视) |
|---|---|---|---|
| `cold_techlead` | 冷酷大厂 Tech Lead | 抠细节 · 不留情面 · 看落地 | 20 / 80 / 70 / 40 |
| `vision_master` | OpenAI 视野流大咖 | 看认知 · 谈格局 · 问哲学 | 50 / 50 / 40 / 95 |
| `product_mentor` | 资深产品 Mentor | 友好引导 · 看潜力 · 重方法论 | 80 / 60 / 30 / 60 |
| `researcher` | 直接的研究员 | 关心原理 · 不耐烦表面 · 技术直球 | 40 / 90 / 60 / 50 |
| `random` | 🎲 随机 | 命运转盘 | 从前 4 张随机抽取,所有滑块 ±15 抖动 |

### 3.3 滑块语义

| 滑块 | 0 端 | 100 端 | 影响 |
|---|---|---|---|
| **温和度** | 冷峻不寒暄 | 温和鼓励 | Interviewer 措辞;Evaluator 不受影响 |
| **深度偏好** | 偏概念/视野 | 偏 hands-on 实操 | 追问方向选择 |
| **节奏** | 慢热 | 高密度连续追问 | 打断阈值、追问频率 |
| **视野** | 抠技术细节 | 拉到行业战略 | 业务情境题的层次 |

### 3.4 交互细节

- 卡片采用 hover 上浮 + active 边框高亮(`--accent`)
- 点卡片后,4 个滑块的值**带过渡动画**滑到该卡片的默认值
- 滑块拖动时**实时显示数字**(0-100),松手时**轻微振动反馈**
- 简历区域用 dashed border + 灰色 placeholder 文字,**不可交互**
- [开始面试] 按钮:点击后 **POST `/api/sessions/:id/start`** 传 persona 配置,拿到确认后跳 `/interview/:sessionId`

## 4. 页面 3:面试房间 `/interview/:sessionId`(MVP 核心)

**目的**:沉浸式的对话面试体验,**这是产品的灵魂页面**。

### 4.1 布局(类 ChatGPT,居中对话流)

```
┌────────────────────────────────────────────────┐
│  [Header 区]                                   │
│  ┌──────────┐  面试官名: 冷酷 Tech Lead         │
│  │  🤔      │  心情: 沉思中...                  │
│  │ 头像/    │  满意度: ████████░░ 65            │
│  │ emoji    │  当前轮: 3 / 至少 5               │
│  └──────────┘  [结束面试] [Debug 切换]          │
├────────────────────────────────────────────────┤
│                                                │
│  [对话流 - 居中 max-width: 720px]              │
│                                                │
│   面试官:                                      │
│   ┌─────────────────────────────────────┐      │
│   │ 你用 Claude Code 时,什么时刻会让   │      │
│   │ 你觉得"这个模型行为有品味"?       │      │
│   │ [▾ 思考过程 (折叠)]                  │      │
│   └─────────────────────────────────────┘      │
│                                                │
│           ┌──────────────────────┐  你:       │
│           │ 我觉得...(打字回答) │            │
│           └──────────────────────┘            │
│                                                │
│   面试官:(打字机效果展示新一题)               │
│   ▊                                            │
│                                                │
├────────────────────────────────────────────────┤
│  [输入区 - 固定底部]                            │
│  ┌──────────────────────────────────────┐      │
│  │ 输入你的回答...                       │      │
│  │                                       │      │
│  └──────────────────────────────────────┘      │
│                              [发送 ↵]          │
└────────────────────────────────────────────────┘
```

### 4.2 Header 区(关键拟人元素)

**头像 emoji 表情**(根据 satisfaction 切换):
| satisfaction 区间 | emoji | 视觉 |
|---|---|---|
| 0-20 | 😑 | 失望 |
| 21-40 | 🤨 | 怀疑 |
| 41-60 | 🤔 | 思考中 |
| 61-80 | 😊 | 满意 |
| 81-100 | 😤 | 锐利专注(高分位反而严肃) |

**进度条**:横向条,根据 satisfaction 渲染宽度;颜色按区间分段(`--bad` / `--mid` / `--good`);**不显示数字**,纯隐喻。

**"心情"/状态文字**(根据 SSE 事件 `interviewer_state` 切换):

10 个拟人状态库(完整清单):
```
1. "面试官正在沉思..."        (Thought 阶段开始)
2. "面试官在敲击桌面..."      (Tool 调用中)
3. "面试官在皱眉..."          (满意度下降)
4. "面试官在点头..."          (满意度上升)
5. "面试官正在记录笔记..."    (Evaluator 阶段)
6. "面试官在翻看资料..."      (Tool: get_skill_tree / standard_answer)
7. "面试官的眼神锐利起来..."  (drill_down 触发)
8. "面试官放下了笔..."        (topic_finish_signal != null)
9. "面试官身体前倾..."        (高满意度持续)
10. "面试官沉默了几秒..."     (用户回答后 LLM 调用中)
```

**状态切换由后端 SSE 事件驱动**(见接口规范),前端只负责渲染。

### 4.3 对话流细节

#### 打字机效果
- 面试官的每条消息**逐字渲染**,速度 30-50ms/字
- SSE 接收 token 流时,**累积到一定长度才 flush 到 DOM**,避免太频繁 re-render
- 渲染中显示**光标闪烁**(▊)

#### 思考过程折叠区
- 每条面试官消息下方有 `[▾ 思考过程]` 按钮
- 默认折叠,点击展开,**显示 ReAct 步骤**:
  ```
  Thought: 用户回答了 X,但漏洞在 Y
  → Tool: query_standard_answer(topic="Agent Loop")
  Observation: 标准答案应该提到 ABC
  Thought: 决定追问 Y 这个方向
  ```
- 用 `font-mono` 字体,灰色背景,小一号字号
- **Debug 模式下默认展开**

#### 用户消息气泡
- 右对齐,主色 `--accent-soft` 背景
- 用户消息**不显示思考过程**

### 4.4 满意度彩蛋(产品感关键)

| 触发 | 彩蛋 |
|---|---|
| satisfaction 从 <80 跨越到 >=80 | **顶部飘下 5-8 条丝带**(Framer Motion + confetti) |
| satisfaction 从 >20 跌到 <=20 | **头像短暂震动 + 弹出小气泡**"面试官失望地摇了摇头" |
| 累积达到第 5 题完成 | 顶部弹出 toast"已达到结束门槛,可以随时结束" |
| `red_flag` 新增 | 头像角标短暂闪现 ❗ 图标 1.5 秒 |
| `bright_spot` 新增 | 头像角标短暂闪现 ✨ 图标 1.5 秒 |

### 4.5 结束面试逻辑

点 [结束面试] 按钮:

- **如果 `topics_covered < 5`**:
  弹窗:
  ```
  ┌─────────────────────────────┐
  │ 您还没回答完 5 道题         │
  │ 当前已答: 3 题              │
  │ 评分数据可能不够准确        │
  │                             │
  │ [继续面试]  [仍然结束]      │
  └─────────────────────────────┘
  ```
  - "继续面试" → 关闭弹窗
  - "仍然结束" → 调 `POST /api/sessions/:id/end` → 跳报告页(报告页提示"早退场,数据有限")

- **如果 `topics_covered >= 5`**:
  直接调 `/end` → 跳 `/report/:sessionId`

### 4.6 输入区

- 多行 textarea,Enter 提交,Shift+Enter 换行
- 字数计数器(右下角小字),超过 500 字时高亮(打断机制阈值,见接口规范)
- 提交后立即清空,显示"面试官沉默了几秒..."状态

## 6. Debug 模式(开发者视角)

### 6.1 激活方式
- URL 加 `?debug=1`
- 或按快捷键 `Cmd+Shift+D` / `Ctrl+Shift+D`
- 激活后顶部 Nav 显示红色 "DEBUG ON" 标识

### 6.2 形态:右侧抽屉(Drawer)
**默认从右侧滑出**,可拖拽宽度(默认 480px),不影响主页面。

### 6.3 分 4 个 Tab

```
┌──────────────────────────────────────────┐
│  Debug Panel              [×]            │
├──────────────────────────────────────────┤
│ [对话流] [ReAct] [Evaluator] [State]    │  ← Tabs
├──────────────────────────────────────────┤
│                                          │
│  (当前 Tab 内容)                          │
│                                          │
└──────────────────────────────────────────┘
```

| Tab | 内容 |
|---|---|
| **对话流** | 主页面对话的原始 JSON 数据(每条消息的完整字段) |
| **ReAct** | 每次 Interviewer/Evaluator 调用的完整 ReAct 步骤树(Thought→Tool→Observe→...) |
| **Evaluator** | 每次 Evaluator 返回的 JSON,**带 diff**(本次 vs 上次,高亮变化字段) |
| **State** | 全局 InterviewState 实时视图,JSON Tree 可展开,**实时随 SSE 更新** |

字体一律用 `font-mono`,语法高亮(JSON syntax highlighting,可用 react-json-view)。

## 8. Mock 数据约定(前端独立开发用)

**为了让前端 Agent 在后端没好之前能独立跑通**,前端必须支持 Mock 模式:

- 环境变量 `VITE_USE_MOCK=true` 启用
- 启用后,所有 API 调用走 `src/api/mock/` 目录的本地数据
- SSE 用定时器模拟事件流
- Mock 数据要**完整覆盖一场 5 题面试的全流程**,事件流真实

Mock 数据的格式严格遵循 [接口规范.md](./接口规范.md) 定义的 schema,**不允许前端自行编造字段**。
