## 0. 约定

| 项 | 值 |
|---|---|
| 后端协议 | HTTP/1.1 + SSE(Server-Sent Events) |
| 后端 base URL(dev) | `http://localhost:8000` |
| 前端 base URL(dev) | `http://localhost:5173` |
| 数据格式 | JSON(`Content-Type: application/json`) |
| 编码 | UTF-8 |
| 时间戳 | ISO 8601 字符串(`2026-06-25T14:30:00Z`) |
| 字段命名 | **snake_case**(后端 Python 主导,前端做转换) |
| sessionId 格式 | UUID v4 字符串 |
| CORS | 开发期允许 `http://localhost:5173` |

### 0.1 通用错误响应

所有错误统一格式:
```json
{
  "error": {
    "code": "INVALID_JD",
    "message": "JD 文本不能为空",
    "details": {}
  }
}
```

| HTTP 状态 | 业务含义 |
|---|---|
| 200 | 成功 |
| 400 | 入参错误 |
| 404 | session 不存在 |
| 409 | session 状态冲突(如已结束的 session 不能继续) |
| 500 | 后端内部错误 |
| 503 | LLM 服务不可用 |

错误码枚举:
```
INVALID_JD             JD 为空或过长
SESSION_NOT_FOUND      sessionId 无效
SESSION_ALREADY_ENDED  会话已结束,不允许继续
PERSONA_INVALID        人格参数不合法
LLM_TIMEOUT            LLM 调用超时
INTERNAL               后端内部错误
```

## 1. REST API 总览

| 方法 | 路径 | 用途 | 阶段 |
|---|---|---|---|
| POST | `/api/sessions` | 创建 session,解析 JD | 首页 |
| GET | `/api/sessions/:id` | 拿 session 元信息 | 准备页/恢复 |
| POST | `/api/sessions/:id/start` | 提交人格配置,启动面试 | 准备页 |
| POST | `/api/sessions/:id/answer` | 提交用户回答(触发 SSE 流) | 面试页 |
| GET | `/api/sessions/:id/stream` | **SSE 长连接**(主推送通道) | 面试页 |
| POST | `/api/sessions/:id/end` | 用户主动结束面试 | 面试页 |
| GET | `/api/sessions/:id/report` | 拿最终报告 | 报告页 |
| GET | `/api/personas` | 拿人格卡片列表(静态) | 准备页 |

### 2.3 `POST /api/sessions/:id/start` — 启动面试

**请求体**:
```json
{
  "persona": {
    "card_id": "cold_techlead",
    "dimensions": {
      "warmth": 20,
      "depth_preference": 85,
      "pace": 75,
      "vision": 45
    }
  }
}
```

**响应**(200):
```json
{
  "session_id": "5f3a8c12-...",
  "status": "ready",
  "stream_url": "/api/sessions/5f3a8c12-.../stream"
}
```

**错误**:
- 404 `SESSION_NOT_FOUND`
- 400 `PERSONA_INVALID`:card_id 不在 personas 列表,或 dimensions 任一不在 0-100

**后端行为**:
- 把 persona 配置写入 session state
- **不立即生成第一题**(等前端连上 SSE 后再开始)
- 把 `persona.dimensions` 注入到 Interviewer 和 Evaluator 的 system prompt(详见 [产品方案.md](../../产品方案.md) §人格矩阵章节)

### 2.4 `GET /api/sessions/:id/stream` — SSE 主通道(★★★ 核心)

**请求**:
- 方法:GET
- Header:`Accept: text/event-stream`
- 前端用 `new EventSource(url)`

**响应**:SSE 流,见 §3 详细事件定义

**连接生命周期**:
- 前端连上后,后端**立即推送第一题**(`event: assistant_message_start`)
- 用户回答前,后端**保持长连接 idle**
- 用户调 `/answer` 后,后端通过 SSE 推送评估和下一题事件
- 用户点结束或所有题完成,后端推 `event: session_ended` 后关闭连接

**重连**:
- 前端断线后自动重连(EventSource 内置)
- 后端按 sessionId 找到会话状态,可以**幂等续推**(不重复推送已经发出的事件)
- 建议前端发 `Last-Event-ID` header,后端按 ID 续接

### 2.5 `POST /api/sessions/:id/answer` — 提交用户回答

**请求体**:
```json
{
  "content": "我觉得 Agent Loop 就是模型不断调用工具直到完成任务的循环..."
}
```

**响应**(200,立即返回):
```json
{
  "accepted": true,
  "turn_id": "t_007"
}
```

**后端行为**:
- 立即返回 200
- **后续所有动作通过 SSE 流推送**(评估开始、思考链、追问/换题等)
- 如果用户回答 > 500 字,后端设 `is_interrupted=true`,下一题带打断引导

**错误**:
- 404 `SESSION_NOT_FOUND`
- 409 `SESSION_ALREADY_ENDED`

---

### 2.6 `POST /api/sessions/:id/end` — 结束面试

**请求体**:
```json
{
  "reason": "user_initiated"   // user_initiated | min_topics_reached | natural_end
}
```

**响应**(200):
```json
{
  "session_id": "5f3a8c12-...",
  "status": "ended",
  "report_url": "/api/sessions/5f3a8c12-.../report",
  "topics_covered_count": 5,
  "below_minimum": false       // 是否低于 5 题门槛
}
```

**说明**:
- 前端**只在用户主动点"结束"且 ≥5 题时直接调**;<5 题时先弹窗,用户确认后再调
- 后端收到后通过 SSE 推 `event: session_ended`,然后关闭连接
- 报告**异步生成**;前端跳到 `/report/:id` 后调 `/report` 接口轮询/等待

## 3. SSE 事件定义(★★★ 核心)

SSE 标准格式:
```
event: <event_name>
data: <json>
id: <event_id>

```

### 3.1 事件总览

| event_name | 触发时机 | 关键字段 |
|---|---|---|
| `session_started` | 连接建立 | session_id, persona |
| `interviewer_state` | 拟人状态变化 | state_text |
| `assistant_message_start` | 新一条面试官消息开始 | message_id, mode |
| `assistant_message_delta` | 打字机增量 token | message_id, delta |
| `assistant_message_end` | 一条消息结束 | message_id, full_text |
| `thinking_step` | ReAct 思考链单步 | message_id, step_type, content |
| `evaluator_started` | 评估开始 | turn_id |
| `evaluator_result` | 评估完成 | turn_id, evaluation |
| `satisfaction_update` | 满意度变化 | new_value, delta |
| `red_flag_added` | 新增红牌 | flag |
| `bright_spot_added` | 新增亮点 | spot |
| `topic_finished` | 本题结束 | topic_id, reason |
| `topic_started` | 新题开始 | topic_id, topic_name, weight |
| `min_topics_reached` | 达到 5 题门槛 | covered_count |
| `error` | 出错 | error_code, message |
| `session_ended` | 面试结束 | reason, report_ready_eta |

### 3.2 事件详细 schema

#### `session_started`
```json
{
  "session_id": "5f3a8c12-...",
  "persona": {
    "card_id": "cold_techlead",
    "name": "冷酷大厂 Tech Lead",
    "dimensions": { "warmth": 20, "depth_preference": 85, "pace": 75, "vision": 45 }
  }
}
```

#### `interviewer_state`
**驱动前端 Header 的"心情"文字切换**。

```json
{
  "state_id": "thinking",       // 见下表枚举
  "state_text": "面试官正在沉思...",
  "ttl_ms": 3000                // 多久后自动切回 idle
}
```

`state_id` 枚举(对应前端设计 §4.2 的 10 个状态):
```
idle                  (默认,无文字 / "等待回答")
thinking              "面试官正在沉思..."
tapping               "面试官在敲击桌面..."
frowning              "面试官在皱眉..."
nodding               "面试官在点头..."
note_taking           "面试官正在记录笔记..."
reading_material      "面试官在翻看资料..."
sharpening_gaze       "面试官的眼神锐利起来..."
pen_down              "面试官放下了笔..."
leaning_forward       "面试官身体前倾..."
silent_pause          "面试官沉默了几秒..."
```

#### `assistant_message_start`
```json
{
  "message_id": "m_023",
  "mode": "OPENING",           // OPENING | DRILL_DOWN | SWITCH_TOPIC | INTERRUPT
  "topic_id": "agent_loop",
  "topic_name": "Agent Loop 与产品形态",
  "depth": 0                   // 当前追问深度
}
```

#### `assistant_message_delta`
**用于打字机效果,逐字推送**。

```json
{
  "message_id": "m_023",
  "delta": "你用 Claude Code "
}
```

> 后端可以选择"按 token 推送"或"按句子推送",前端用 30-50ms/字的速度渲染。建议**按 token 推送但有最小间隔限制**(避免过频)。

#### `assistant_message_end`
```json
{
  "message_id": "m_023",
  "full_text": "你用 Claude Code 时,什么时刻会让你觉得'这个模型行为有品味'?",
  "thinking_steps_count": 4
}
```

#### `thinking_step`
**ReAct 单步,用于折叠区展示**。

```json
{
  "message_id": "m_023",
  "step_index": 0,
  "step_type": "thought",      // thought | tool_call | observation | final
  "content": "用户上一次回答缺乏具体场景,我需要追问 first_hand 维度",
  "tool_name": null,           // step_type=tool_call 时有
  "tool_args": null
}
```

工具调用 example:
```json
{
  "message_id": "m_023",
  "step_index": 1,
  "step_type": "tool_call",
  "tool_name": "get_red_flags_history",
  "tool_args": {}
}
```

```json
{
  "message_id": "m_023",
  "step_index": 2,
  "step_type": "observation",
  "content": "[{\"topic\":\"Agent Loop\",\"evidence\":\"无第一手实践\",\"severity\":\"medium\"}]"
}
```

#### `evaluator_started`
```json
{ "turn_id": "t_007" }
```
**前端反应**:把"心情"切到 `note_taking`。

#### `evaluator_result`
**评估完整结果,Debug 模式显示**。

```json
{
  "turn_id": "t_007",
  "evaluation": {
    "answer_quality": "surface",
    "dimension_deltas": {
      "agent_product_taste": 0,
      "tech_understanding": -3,
      "product_methodology": 0,
      "user_empathy": 0,
      "first_hand_experience": -5
    },
    "current_satisfaction": 35,
    "found_gaps": ["未给出具体场景"],
    "found_strengths": [],
    "drill_down_target": "first_hand_experience",
    "drill_down_hint": "追问其具体使用 Claude Code 的某一个真实场景",
    "red_flags_to_add": [
      { "topic": "Agent Loop", "evidence": "无第一手实践", "severity": "medium" }
    ],
    "bright_spots_to_add": [],
    "topic_finish_signal": null
  }
}
```

#### `satisfaction_update`
```json
{
  "topic_id": "agent_loop",
  "previous_value": 50,
  "new_value": 35,
  "delta": -15
}
```
**前端反应**:进度条动画,emoji 头像切换;跨越阈值时触发彩蛋(80↑ 丝带 / 20↓ 震动)。

#### `red_flag_added`
```json
{
  "flag": { "topic": "Agent Loop", "evidence": "无第一手实践", "severity": "medium" }
}
```
**前端反应**:头像角标闪现 ❗ 1.5 秒。

#### `bright_spot_added`
```json
{
  "spot": { "topic": "MCP", "evidence": "提到具体的 stdio transport 细节" }
}
```
**前端反应**:头像角标闪现 ✨ 1.5 秒。

#### `topic_finished`
```json
{
  "topic_id": "agent_loop",
  "reason": "satisfaction_reached",  // satisfaction_reached | depth_limit | user_hopeless | user_excellent | exhausted
  "final_satisfaction": 82
}
```

#### `topic_started`
```json
{
  "topic_id": "tool_use",
  "topic_name": "Tool Use 与 MCP",
  "weight": 2.0,
  "depth_level": "medium",
  "is_red_flag_hunt": false   // 是否是针对 red_flag 的挖坑题
}
```

#### `min_topics_reached`
```json
{ "covered_count": 5 }
```
**前端反应**:顶部弹 toast"已达到结束门槛,可以随时结束"。

#### `error`
```json
{
  "error_code": "LLM_TIMEOUT",
  "message": "LLM 调用超时,请稍候重试",
  "recoverable": true
}
```

#### `session_ended`
```json
{
  "reason": "user_initiated",       // user_initiated | natural_end | error
  "report_ready_eta_ms": 5000,
  "below_minimum": false
}
```

### 3.3 事件流的典型时序

**一次"用户提交回答 → 追问"完整时序**:

```
[用户调 POST /answer]
   ↓
event: interviewer_state         (silent_pause)
event: evaluator_started
event: thinking_step             (Evaluator ReAct)
event: thinking_step             (Tool call: query_standard_answer)
event: thinking_step             (Observation)
event: thinking_step             (Final)
event: evaluator_result          (完整 JSON)
event: satisfaction_update       (35)
event: red_flag_added            (如果有)
   ↓
event: interviewer_state         (thinking)
event: assistant_message_start   (mode=DRILL_DOWN, depth=1)
event: thinking_step             (Interviewer ReAct)
...
event: assistant_message_delta   (×N)
event: assistant_message_end
event: interviewer_state         (idle)
```

**一次"题目结束 → 换题"时序**:

```
event: evaluator_result          (satisfaction=82)
event: satisfaction_update       (82, ↑ 触发丝带彩蛋)
event: topic_finished            (reason=satisfaction_reached)
event: interviewer_state         (pen_down)
event: topic_started             (新题)
event: interviewer_state         (thinking)
event: assistant_message_start   (mode=OPENING, depth=0)
...
```

**5 题门槛事件**:

```
event: topic_finished            (第 5 题)
event: min_topics_reached        (covered_count=5)
event: topic_started             (第 6 题)
... 继续直到用户主动结束 ...
```

## 4. 数据 Schema 汇总(供前端 TypeScript 类型生成)

```typescript
// types/api.ts

export type SessionId = string;
export type MessageId = string;
export type TurnId = string;

export interface PersonaDimensions {
  warmth: number;          // 0-100
  depth_preference: number;
  pace: number;
  vision: number;
}

export interface Persona {
  card_id: string;
  name?: string;
  tags?: string[];
  description?: string;
  default_dimensions?: PersonaDimensions;
}

export interface SessionStartRequest {
  persona: {
    card_id: string;
    dimensions: PersonaDimensions;
  };
}

export interface SkillTreeNode {
  id: string;
  topic: string;
  weight: number;
  depth_level: 'shallow' | 'medium' | 'deep';
  dimensions: string[];
}

export interface SkillTree {
  root: string;
  dimensions: string[];
  nodes: SkillTreeNode[];
}

export interface RedFlag {
  topic: string;
  evidence: string;
  severity: 'low' | 'medium' | 'high';
}

export interface BrightSpot {
  topic: string;
  evidence: string;
}

export interface EvaluatorResult {
  answer_quality: 'surface' | 'concept' | 'hands_on' | 'insight';
  dimension_deltas: Record<string, number>;
  current_satisfaction: number;
  found_gaps: string[];
  found_strengths: string[];
  drill_down_target: string | null;
  drill_down_hint: string | null;
  red_flags_to_add: RedFlag[];
  bright_spots_to_add: BrightSpot[];
  topic_finish_signal: 'exhausted' | 'user_excellent' | 'user_hopeless' | null;
}

export type InterviewerMode = 'OPENING' | 'DRILL_DOWN' | 'SWITCH_TOPIC' | 'INTERRUPT';
```

## 5. 前端 SSE 接入参考代码(给前端 Agent)

```typescript
// src/api/sse.ts
import { useEffect } from 'react';
import { useInterviewStore } from '@/store/interviewStore';

export function useInterviewStream(sessionId: string) {
  const store = useInterviewStore();

  useEffect(() => {
    const url = `${import.meta.env.VITE_API_BASE}/api/sessions/${sessionId}/stream`;
    const es = new EventSource(url);

    es.addEventListener('assistant_message_start', (e) => {
      store.startMessage(JSON.parse(e.data));
    });
    es.addEventListener('assistant_message_delta', (e) => {
      store.appendDelta(JSON.parse(e.data));
    });
    es.addEventListener('assistant_message_end', (e) => {
      store.endMessage(JSON.parse(e.data));
    });
    es.addEventListener('thinking_step', (e) => {
      store.addThinkingStep(JSON.parse(e.data));
    });
    es.addEventListener('evaluator_result', (e) => {
      store.setEvaluation(JSON.parse(e.data));
    });
    es.addEventListener('satisfaction_update', (e) => {
      store.updateSatisfaction(JSON.parse(e.data));
    });
    es.addEventListener('interviewer_state', (e) => {
      store.setInterviewerState(JSON.parse(e.data));
    });
    es.addEventListener('red_flag_added', (e) => store.addRedFlag(JSON.parse(e.data)));
    es.addEventListener('bright_spot_added', (e) => store.addBrightSpot(JSON.parse(e.data)));
    es.addEventListener('topic_finished', (e) => store.finishTopic(JSON.parse(e.data)));
    es.addEventListener('topic_started', (e) => store.startTopic(JSON.parse(e.data)));
    es.addEventListener('min_topics_reached', (e) => store.markMinReached(JSON.parse(e.data)));
    es.addEventListener('session_ended', (e) => store.endSession(JSON.parse(e.data)));
    es.addEventListener('error', (e) => store.setError(JSON.parse(e.data)));

    es.onerror = () => store.setSseStatus('disconnected');
    es.onopen = () => store.setSseStatus('connected');

    return () => es.close();
  }, [sessionId]);
}
```

## 6. 后端 SSE 推送参考(给后端 Agent)

后端用 **FastAPI + StreamingResponse + LangGraph stream**:

```python
# api/streams.py
from fastapi import APIRouter
from fastapi.responses import StreamingResponse
import json, asyncio

router = APIRouter()

async def event_generator(session_id: str):
    session = get_session(session_id)
    async for event in session.run_graph_stream():
        yield format_sse(event)

def format_sse(event: dict) -> str:
    name = event["event"]
    data = json.dumps(event["data"], ensure_ascii=False)
    eid = event.get("id", "")
    return f"event: {name}\nid: {eid}\ndata: {data}\n\n"

@router.get("/api/sessions/{session_id}/stream")
async def stream(session_id: str):
    return StreamingResponse(
        event_generator(session_id),
        media_type="text/event-stream",
        headers={"Cache-Control": "no-cache", "X-Accel-Buffering": "no"}
    )
```
